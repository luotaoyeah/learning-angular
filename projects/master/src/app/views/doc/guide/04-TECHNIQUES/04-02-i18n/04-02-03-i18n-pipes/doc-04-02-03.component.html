angular 提供了几个 pipe 用来帮助实现 i18n: DatePipe, CurrencyPipe, DecimalPipe, PercentagePipe

angular 默认只包含了 en-US 的 locale data, 如果我们要设置其他的 locale, 比如 zh-Hans, 我们需要引入对应的 locale data, angular 所有支持的 locale data 都可以在 @angular/common/locales 下面找到, 比如: @angular/common/locales/zh, @angular/common/locales/en

如果我们不引入对应的 locale data, 那么我们在使用 DatePipe 这些国际化相关的 pipe 时, 就不能使用这些对应的 locale 数据, 就会报错

locale data 里面包含的数据, 就是一些日期/时间/货币等等的本地化数据, 比如中文里面的年/月/日/星期, 英文里面的 year/month/day/week

从 @angular/common/locales/zh 引入 locale data 之后, 我们需要调用 registerLocaleData() 方法来注册这些 locale data

这些 locale data 会使用它们里面定义的默认的 locale id, 但是我们也可以通过 registerLocaleDate() 方法的第二个参数, 重新设置一个新的 locale id
