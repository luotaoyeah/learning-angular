class binding 跟普通的 property binding 类似, 但是它们之间有一些区别, 我们知道对于 property binding 来说, 如果使用的是 someProperty="someValue" 的形式, 则首先会去匹配对应的 directive input property, 然后会去匹配对应的 element attribute

而对于 class binding 来说, class 实际上是一个 attribute, 它对应的 properrty 是 className, 所以当我们使用 class="someClass" 的形式时, 实际上绑定的就是对应的 attribute

FOO

因为 class 是一个 attribute, 因此我们可以使用 attribute bingding 的语法来实现 class binding, 此时, 如果同时存在 class="x y z", 则它会被覆盖, 无论谁先谁后

FOO

可以使用 [class]="'foo bar baz'" 的形式来动态地绑定多个类名, 此时, 如果同时存在 class="x y z", 则它会被覆盖, 无论谁先谁后

FOO

可以使用 [class.foo]="isFoo" 的形式来动态的添加移除一个类名, 此时, 如果同时存在 class="x y z", 则它**不**会被覆盖, 会跟 [class.foo]="isFoo" 的结果进行合并
