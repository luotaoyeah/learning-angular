component 按照加载方式可以分为两类：

  • 声明式的（declarative）component
  • 命令式的（imperative）component

所谓 declarative component 指的是：直接在 template 中使用的组件，如：<comp01></comp01>

所谓 imperative component 指的是：通过命令式的方式，比如调用方法，动态创建的 component

imperative component 必须在 NgModule 的 metadata 中的 entryComponents 配置项中声明，所以，一般把 imperative component 称之为 entry component

entry component 有三种：

  • 通过 bootstrap 声明的 component，也称之为 bootstraped component
  • 通过 route definition 声明的 component
  • 显式地在 entryComponents 中声明的 component

bootstraped component 和 router component 不需要显示地在 entryComponents 中声明，因为 angular 会自动将它们添加到 entryComponents 中去

显示地在 entryComponents 中声明的 entry component，一般是用来进行动态加载的 component

angular 的 compiler 在编译的时候，会将没有被使用到的 component 排除掉，也就是说，一个 component 即使在 declarations 中声明了，但是如果它不是一个 entry component，同时它也没有在其他的 template 中被使用，那么它就不会被包含在最终打包的代码中去
