根据前面的经验, 当使用 [someProperty]="someExpression" 的形式时, 会首先去匹配 directive input property, 然后去匹配 element property, 当都匹配不到时, 就会报错

当使用 someProperty="someValue" 的形式时, 会首先去匹配 directive input property, 然后去匹配 element attribute, 这时候, 有两种情况:

  • 如果匹配不到一个名为 someProperty 的 attribute, 则 someProperty 会成为一个自定义的 attribute
  • 如果真的存在一个名为 someProperty 的 attribute, 则会报错, 提示不能使用 property binding 的语法来绑定 attribute

如何才能正确地绑定 HTML attribute 呢? 需要使用一种特殊的语法形式 [attr.some-attribute]="expression"
