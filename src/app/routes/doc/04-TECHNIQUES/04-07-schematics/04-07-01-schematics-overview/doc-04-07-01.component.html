schematics 是如何打包的? 通常将多个 schematics 打包到一个 schematic collection 中, 例如: @schematics/angular 是一个 collection, 它里面有很多 schematics(component/service/directive)

如何使用某个 schematic? 通过 ng generate 和 ng add 和 ng update 这三个命令来使用某个 schematic, 命令格式为: ng generate <schematic-collection>:<schematic>, 例如: ng generate @schematics/angular:component

默认的 collection 为 @schematics/angular 如果没有指定 collection, 则默认的 collection 为 @schematics/angular, 因此平时我们在使用诸如 ng g component some-comp 这样的命令时, 就不需要指定 collection

如何配置 schematics 的参数的默认值? 在 angular.json#schematics 和 angular.json#projects.learning-angular.schematics 中, 可以对 schematics 的参数的默认值进行配置, 前者是针对整个 workspace 的, 后者是针对某个 project 的
